 Description   
Currently all .txt and .xml and interpolated, and nothing else. This should be specified, possibly via a top level 'interpolation.properties' file that can somehow be specified for the packaging up.

Some kind of flag will be needed?
Comment by bayard [ Wed, 4 Feb 2004 13:04:17 -0800 (PST) ]
If the first file is a *.properties [payload.properties], and it contains a line of org.osjava.payload=true, then it is promoted to the top level and not included in the payload itself.

It can have a line of:

org.osjava.payload.interpolate.match=txt,xml,AlertGui.jnlp

etc.
 No wildcarding, just String matching. Maybe allow regexps later on.
Comment by bayard [ Wed, 4 Feb 2004 22:51:58 -0800 (PST) ]
org.osjava.payload=true
org.osjava.payload.interpolate.endsWith=txt
org.osjava.payload.interpolate.endsWith=xml
org.osjava.payload.interpolate.matches=regexp

shows payload.properties possibilities. Top line is how we know it's a special file.
