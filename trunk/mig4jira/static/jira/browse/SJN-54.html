[SJN-54] If a resource directory config/java and file config/java.properties both exist, indeterminate result

Type:BugPriority: Major
Reporter:Dan GreeningAssignee:Henri Yandell
Resolution:Fixed 
Environment:


 Description   
If you have a file

config/
         java.properties

Where java.properties defines a sub-JNDI object such as java.thongDS

and a file

config/
         java/
              thingDS.properties

Simple-JNDI will never see the config/java.properties file.

This could be fixed by documenting it or checking for the existance of the java.properties file and spewing a warning.

Or it could be fixed so both could coexist well.
Comment by bayard [ Wed, 22 Sep 2004 19:47:03 -0700 (PDT) ]
Documentation seems the easy way to go.

Ideally, it should use both except when there is a complete key clash, in which case one should win and be documented that way.
